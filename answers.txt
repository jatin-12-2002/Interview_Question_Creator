**1. Main Table: `sdgs`**

* **`goal_id` (INT, PRIMARY KEY):** Unique identifier for each SDG (e.g., 1, 2, 3, ... 17).
* **`goal_name` (VARCHAR):** The name of the SDG (e.g., "No Poverty", "Zero Hunger").
* **`goal_description` (TEXT):** A detailed description of the SDG's objectives.
* **`target_year` (INT):** The year by which the SDG aims to be achieved (e.g., 2030).

**2. Table: `targets`**

* **`target_id` (INT, PRIMARY KEY):** Unique identifier for each target within an SDG.
* **`goal_id` (INT, FOREIGN KEY REFERENCES `sdgs`(`goal_id`)):** Links the target to its corresponding SDG.
* **`target_description` (TEXT):** A detailed description of the specific target.
* **`target_indicator` (VARCHAR):** A specific indicator used to measure progress towards the target.

**3. Table: `indicators`**

* **`indicator_id` (INT, PRIMARY KEY):** Unique identifier for each indicator.
* **`target_id` (INT, FOREIGN KEY REFERENCES `targets`(`target_id`)):** Links the indicator to its corresponding target.
* **`indicator_name` (VARCHAR):** The name of the indicator (e.g., "Proportion of population living below the international poverty line").
* **`indicator_definition` (TEXT):** A detailed definition of how the indicator is measured.
* **`data_source` (VARCHAR):** The source of data for the indicator (e.g., World Bank, UN Data).
* **`data_type` (VARCHAR):** The type of data (e.g., percentage, number, ratio).

**4. Table: `progress`**

* **`progress_id` (INT, PRIMARY KEY):** Unique identifier for each progress record.
* **`indicator_id` (INT, FOREIGN KEY REFERENCES `indicators`(`indicator_id`)):** Links the progress record to its corresponding indicator.
* **`year` (INT):** The year for which the progress data is recorded.
* **`value` (DECIMAL):** The actual value of the indicator for that year.

**5. Table: `countries`**

* **`country_id` (INT, PRIMARY KEY):** Unique identifier for each country.
* **`country_name` (VARCHAR):** The name of the country.
* **`iso_code` (VARCHAR):** The ISO 3166-1 alpha-3 code for the country.

**6. Table: `country_progress`**

* **`country_progress_id` (INT, PRIMARY KEY):** Unique identifier for each country's progress record.
* **`country_id` (INT, FOREIGN KEY REFERENCES `countries`(`country_id`)):** Links the progress record to its corresponding country.
* **`indicator_id` (INT, FOREIGN KEY REFERENCES `indicators`(`indicator_id`)):** Links the progress record to its corresponding indicator.
* **`year` (INT):** The year for which the progress data is recorded.
* **`value` (DECIMAL):** The actual value of the indicator for that country and year.

**Relationships:**

* **`sdgs`** has a one-to-many relationship with **`targets`**.
* **`targets`** has a one-to-many relationship with **`indicators`**.
* **`indicators`** has a one-to-many relationship with **`progress`**.
* **`countries`** has a many-to-many relationship with **`indicators`** through **`country_progress`**.

**Benefits of this Structure:**

* **Organized Data:**  The database effectively separates and organizes information about SDGs, targets, indicators, and progress.
* **Flexibility:**  The structure allows for easy addition of new SDGs, targets, indicators, and progress data as needed.
* **Analysis:**  The database facilitates analysis of progress towards each SDG, both globally and at the country level.

## Question:
   Write a Python function that takes a list of daily incomes for a population and calculates the percentage of people living below the extreme poverty line of $1.25 per day.

### Answer:
Certainly! Below is a Python function that takes a list of daily incomes for a population and calculates the percentage of people living below the extreme poverty line of $1.25 per day.

```python
def calculate_poverty_percentage(daily_incomes):
    """
    Calculate the percentage of people living below the extreme poverty line of $1.25 per day.

    Parameters:
    daily_incomes (list of float): A list of daily incomes for a population.

    Returns:
    float: The percentage of people living below the extreme poverty line.
    """
    poverty_line = 1.25
    count_below_poverty_line = sum(1 for income in daily_incomes if income < poverty_line)
    total_people = len(daily_incomes)

    if total_people == 0:
        return 0.0  # Avoid division by zero

    poverty_percentage = (count_below_poverty_line / total_people) * 100
    return poverty_percentage

# Example usage:
daily_incomes = [1.0, 2.5, 0.5, 1.2, 3.0, 0.8, 1.5]
poverty_percentage = calculate_poverty_percentage(daily_incomes)
print(f"Percentage of people living below the extreme poverty line: {poverty_percentage:.2f}%")
```

### Explanation:
1. **Function Definition**: The function `calculate_poverty_percentage` takes a list of daily incomes as input.
2. **Poverty Line**: The extreme poverty line is set to $1.25.
3. **Count Below Poverty Line**: The function counts how many incomes are below the poverty line using a generator expression.
4. **Total People**: The total number of people is determined by the length of the `daily_incomes` list.
5. **Avoid Division by Zero**: If the list is empty, the function returns 0.0 to avoid division by zero.
6. **Calculate Percentage**: The percentage of people living below the poverty line is calculated and returned.

### Example Usage:
The example usage demonstrates how to call the function with a sample list of daily incomes and print the resulting percentage.

**Sustainable Agriculture** is a farming approach that meets society's current food and textile needs without compromising the ability of future generations to meet their own needs. It integrates three main goals: environmental stewardship, economic profitability, and social and economic equity.

1. **Precision Agriculture**: This involves using technologies like GPS, GIS, and remote sensing to optimize field-level management of crops. It helps farmers apply inputs like fertilizers, pesticides, and water more precisely, reducing waste and environmental impact.

   *Implementation in Developing Countries*: Investment in affordable GPS-enabled devices and open-source GIS software, along with training programs for farmers. Governments and NGOs can provide subsidies and establish demo farms to promote adoption.

2. **Vertical Farming**: This is the practice of growing crops in vertically stacked layers, often using artificial light and controlled environments. It saves land and water, and reduces the need for pesticides.

   *Implementation in Developing Countries*: Establish public-private partnerships to build vertical farms in urban areas. Provide training and microfinancing to help local entrepreneurs start their own vertical farming businesses.

3. **Agroforestry Technologies**: Integrating trees with crops and livestock can improve soil health, sequester carbon, and increase biodiversity. Technologies like silvopasture (combining trees, livestock, and improved pasture) and alley cropping (growing crops between rows of trees) can increase farm productivity.

   *Implementation in Developing Countries*: Promote agroforestry practices through farmer field schools and extension services. Provide incentives like payments for ecosystem services to encourage adoption.

**Combating Hunger and Malnutrition**: These innovations can help combat hunger and malnutrition by:

- **Increasing Crop Yields**: Precision agriculture and vertical farming can increase the amount of food produced per acre, making more food available.
- **Improving Nutrition**: Agroforestry can diversify diets by providing fruits, nuts, and other nutritious foods along with crops.
- **Enhancing Farmers' Incomes**: By adopting these technologies, farmers can reduce costs, increase profits, and improve their families' access to food.
- **Building Resilience**: These practices can make farms more resilient to climate change, pests, and diseases, ensuring more stable food supplies.

To make these innovations accessible to smallholder farmers in developing countries, it's crucial to provide affordable financing, quality training, and supportive policies. Collaboration between governments, NGOs, and the private sector can help scale up these technologies and combat hunger and malnutrition effectively.

   - Establish water quality testing protocols and regularly monitor water sources and distribution points for potential contaminants.
   - Implement a real-time water quality monitoring system using sensors and IoT devices to detect and address issues promptly.

   - Develop a fair and efficient water rationing system based on household size, needs, and available supply.

   - Establish water distribution points and schedules, and communicate them effectively to the community.

5. **Data Management and Information System:**
   - Develop a centralized database to store and manage data on water resources, infrastructure, quality, distribution, and consumption.

   - Launch awareness campaigns to educate the community about water conservation, hygiene, and the importance of protecting water sources.

8. **Monitoring, Evaluation, and Adaptation:**
   - Regularly monitor and evaluate the system's performance using relevant indicators (e.g., water availability, quality, access, and consumption).

   - Adapt and update the system as needed, based on changing conditions, new technologies, and lessons learned.

9. **Financial Sustainability:**
   - Develop a sustainable financing strategy, combining government funding, user fees, and external support (e.g., donors, NGOs).
   - Ensure that water tariffs are affordable for the poorest households while generating enough revenue to maintain and expand the system.
   - Explore innovative financing mechanisms, such as public-private partnerships and blended finance.

By integrating these key components and strategies, you can create a sustainable and effective water management system tailored to the region's specific needs and challenges.

## Question:
   Imagine you are developing a web application to raise awareness about the Sustainable Development Goals. Describe the architecture of this application, including the technologies you would use for the frontend, backend, and database. Additionally, explain how you would ensure the application is accessible to users in developing countries with limited internet connectivity.

### Answer:
### Web Application Architecture for Raising Awareness about the Sustainable Development Goals

#### Frontend
**Technologies:**
- **HTML5/CSS3:** For structuring and styling the web pages.
- **JavaScript/TypeScript:** For adding interactivity and dynamic content.
- **React.js:** A popular JavaScript library for building user interfaces, especially single-page applications.
- **Next.js:** A React framework that enables server-side rendering and static site generation, which can improve performance and SEO.

**Features:**
- **Responsive Design:** Ensure the application is accessible on various devices, including mobile phones, tablets, and desktops.
- **Progressive Web App (PWA):** Use service workers and caching strategies to allow offline access and improve performance in low-connectivity areas.
- **Accessibility:** Implement ARIA roles, keyboard navigation, and screen reader support to make the application accessible to users with disabilities.

#### Backend
**Technologies:**
- **Node.js:** A JavaScript runtime for building scalable network applications.
- **Express.js:** A minimal and flexible Node.js web application framework.
- **GraphQL:** A query language for APIs that allows clients to request exactly the data they need.
- **Apollo Server:** A GraphQL server that integrates with Express.js.

**Features:**
- **RESTful API:** For traditional endpoints if needed.
- **GraphQL API:** For more efficient data fetching.
- **Authentication and Authorization:** Implement JWT (JSON Web Tokens) for secure user authentication.
- **Caching:** Use Redis or another in-memory data store to cache frequently accessed data and reduce server load.

#### Database
**Technologies:**
- **MongoDB:** A NoSQL database that is flexible and scalable, suitable for handling diverse data types.
- **Mongoose:** An ODM (Object Data Modeling) library for MongoDB and Node.js.

**Features:**
- **Schema Design:** Define schemas for users, SDGs, events, donations, etc.
- **Indexing:** Optimize queries by indexing frequently accessed fields.
- **Backup and Recovery:** Implement regular backups and a recovery plan to ensure data integrity.

#### Ensuring Accessibility in Developing Countries
**Strategies:**
- **Progressive Web App (PWA):** As mentioned, PWAs can work offline and provide a native app-like experience, which is crucial in areas with limited connectivity.
- **Content Delivery Network (CDN):** Use a CDN to distribute content closer to users, reducing latency and improving load times.
- **Data Compression:** Compress images and other media to reduce data usage.
- **Lazy Loading:** Load content only when it is needed to improve initial load times.
- **Offline-First Approach:** Design the application to work offline first, with synchronization when connectivity is available.
- **Localization:** Provide content in multiple languages to cater to a global audience.
- **Low-Bandwidth Optimization:** Optimize the application to work efficiently on low-bandwidth connections, such as by reducing the size of assets and minimizing data transfers.

### Summary
The architecture of the web application would involve a responsive and accessible frontend built with React.js and Next.js, a robust backend using Node.js and Express.js with GraphQL for efficient data fetching, and a scalable database using MongoDB. To ensure accessibility in developing countries, the application would be designed as a Progressive Web App, use a CDN for content delivery, compress data, implement lazy loading, and adopt an offline-first approach. These strategies would help in providing a seamless experience even in areas with limited internet connectivity.
